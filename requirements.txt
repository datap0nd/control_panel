Rank each account within the current row context

Source data columns, starting from column C:
C = Channel
D = Customer / Country
E = OrgSales / Account
F = Biz Group
G = Seg 1
H = Plan
I = Category / Year
J = 1 Q.
K = Total
L = Value / Sales
M = 2 Q.

Use this as a helper formula in a new column, for example N2.
It does NOT return the top account name.
It returns the rank of the current row's account, from E2, within
the current row's context.

The context used by the formula is:
C = this row's Channel
D = this row's Customer / Country
F = this row's Biz Group
H = this row's Plan
G = hard-filtered to Sub Total
I = hard-filtered to 2026 for ranking

So if the current row is account ABC inside:
Channel = 2026
Customer = IRAN
Biz Group = SMART
Plan = whatever is in H2
Seg 1 = Sub Total
Category = 2026

then the formula returns ABC's rank within that exact context.

Formula for N2:

=LET(
  norm,LAMBDA(x,TRIM(SUBSTITUTE(x&"",CHAR(160)," "))),
  num,LAMBDA(x,IFERROR(1*SUBSTITUTE(SUBSTITUTE(SUBSTITUTE(x&"",",",""),"$","")," ",""),0)),
  ch,norm($C2),
  ctry,norm($D2),
  acct,norm($E2),
  biz,norm($F2),
  plan,norm($H2),
  chanCol,$C$2:$C$100000,
  custCol,$D$2:$D$100000,
  acctCol,$E$2:$E$100000,
  bizCol,$F$2:$F$100000,
  segCol,$G$2:$G$100000,
  planCol,$H$2:$H$100000,
  yearCol,$I$2:$I$100000,
  valCol,$L$2:$L$100000,
  sales,num(valCol),
  ctx,(norm(chanCol)=ch)*
      (norm(custCol)=ctry)*
      (norm(bizCol)=biz)*
      (norm(planCol)=plan)*
      (norm(segCol)="Sub Total")*
      (norm(yearCol)="2026")*
      (norm(acctCol)<>""),
  accts,UNIQUE(FILTER(norm(acctCol),ctx)),
  totals,BYROW(accts,LAMBDA(a,SUM(FILTER(sales,ctx*(norm(acctCol)=a),0)))),
  myTotal,SUM(FILTER(sales,ctx*(norm(acctCol)=acct),0)),
  IFERROR(1+SUM(--(totals>myTotal)),"")
)

How the rank works:
1. The formula reads the current row's account from E2.
2. It finds all rows in the same context: same Channel, Customer,
   Biz Group, and Plan.
3. It then forces Seg 1 to Sub Total and Category / Year to 2026,
   because the rank should be based on 2026 subtotal sales.
4. It sums 2026 Value / Sales by account inside that context.
5. It compares the current account's 2026 sales to the other account
   totals. Rank 1 means no account is higher. Rank 3 means two account
   totals are higher.

You can then use this helper rank as another SUMIFS criterion.
For example, if the helper rank is in column N, rank 3 can be filtered
with:

$N:$N,3

Notes:
1. If Plan should not be part of the context, remove these two parts:
   plan,norm($H2),
   (norm(planCol)=plan)*
2. If Seg 1 should come from the row instead of always Sub Total,
   replace:
   (norm(segCol)="Sub Total")*
   with:
   (norm(segCol)=norm($G2))*
3. If your data goes beyond row 100000, extend all $2:$100000 ranges.
4. Ties receive the same rank. For example, if two accounts tie for
   rank 2, the next account will be rank 4.
