Rank each OrgSales account within the current row context

Headers start in row 3, beginning in column C:
C = Channel
D = Customer / Country
E = OrgSales / Account
F = Biz Group
G = Seg1
H = Plan
I = Category / Year
J = 1 Q.
K = Total
L = Value / Sales
M = 2 Q.

Put this helper formula in N4 and copy it down.

What it does:
It returns the rank of the current row's OrgSales account from E4.
The ranking is calculated within the current row's Channel, Customer,
and Biz Group.

Hard filters:
G / Seg1 = Sub Total
I / Category = 2026

Ignored fields:
H / Plan
J / 1 Q.
K / Total
M / 2 Q.

Formula for N4:

=IFERROR(LET(
  norm,LAMBDA(x,TRIM(SUBSTITUTE(x&"",CHAR(160)," "))),
  num,LAMBDA(x,IFERROR(1*SUBSTITUTE(SUBSTITUTE(SUBSTITUTE(x&"",",",""),"$","")," ",""),0)),
  ch,norm($C4),
  ctry,norm($D4),
  acct,norm($E4),
  biz,norm($F4),
  chanCol,$C$4:$C$100000,
  custCol,$D$4:$D$100000,
  acctCol,$E$4:$E$100000,
  bizCol,$F$4:$F$100000,
  segCol,$G$4:$G$100000,
  yearCol,$I$4:$I$100000,
  valCol,$L$4:$L$100000,
  sales,num(valCol),
  ctx,(norm(chanCol)=ch)*
      (norm(custCol)=ctry)*
      (norm(bizCol)=biz)*
      (norm(segCol)="Sub Total")*
      (norm(yearCol)="2026")*
      (norm(acctCol)<>""),
  accts,UNIQUE(FILTER(norm(acctCol),ctx)),
  totals,BYROW(accts,LAMBDA(a,SUM(FILTER(sales,ctx*(norm(acctCol)=a),0)))),
  myTotal,INDEX(totals,XMATCH(acct,accts,0)),
  1+SUM(--(totals>myTotal))
),"")

Logic:
1. Read the current row context:
   Channel from C4, Customer from D4, Biz Group from F4.
2. Read the current row account:
   OrgSales from E4.
3. Build the ranking pool:
   all rows with the same Channel, Customer, and Biz Group,
   where Seg1 is Sub Total and Category is 2026.
4. Ignore Plan completely.
5. Group that ranking pool by OrgSales.
6. Sum Value from column L for each OrgSales account.
7. Rank the current row's OrgSales by descending summed Value.
   Rank 1 means no account has a higher Value.
   Rank 3 means two accounts have a higher Value.

You can then use column N as another SUMIFS criterion.
For example:

$N:$N,3

Notes:
1. This treats OrgSales as the account being ranked.
   Do not add OrgSales as a context filter before ranking, or each
   account only compares against itself.
2. If your data goes beyond row 100000, extend all $4:$100000 ranges.
3. Ties receive the same rank. For example, if two accounts tie for
   rank 2, the next account will be rank 4.
